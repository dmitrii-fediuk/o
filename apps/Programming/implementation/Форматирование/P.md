Требования к твоему программному коду // Общие

# 1.
Никогда не пиши в коде свои комментарии.

# 2.
Никогда не убирай из кода уже имеющиеся там комментарии.

# 3.
Никогда не пиши в коде полностью пустые строки.

# 4.
## 4.1.
Identation твоего нового кода всегда делай с использованием табуляции вместо пробелов (если синтаксис языка программирования это позволяет).
## 4.2.
При этом Identation кода, который существовал до тебя (был тебе дан как исходный) — не меняй.

# 5.
У любой функции должно быть не более одного return point.

# 6.
В операциях сопоставления значений переменной и константы константа всегда должна быть расположена слева от переменной (если синтаксис языка программирования это позволяет).

# 7. Условное ветвление алгоритма (`if` / `else`)
## 7.1. Общее правило
Общее правило: более короткие (по количеству строк) условные блоки всегда размещай выше более длинных.

## 7.2. Пример
### 7.2.1.
Пусть дан код:
```code
if (A) {
	BLOCK_1
}
else {
	BLOCK_2
}
```

## 7.2.2. Порядок действий
- Пусть `С1` — количество строк кода в `BLOCK_1`.
- Пусть `С2` — количество строк кода в `BLOCK_2`.
- Если `С1` > `C2` — значит, `BLOCK_1` надо переместить из блока `if` в блок `else` (разумеется, инвертировав при этм условие `A`).
- Если `С1` = 0 или `C2` = 0, то соответствующий пустой блок создавать не нужно

# 8.
## 8.1.
Строковые константы всегда по возможности обрамляй одинарными кавычами (') вместо двойных (").
## 8.2.
Используй двойные кавычки только в тех случаях, когда они обеспечивают какое-либо конкретное преимущество перед одинарными.
Например, в некоторых языках программирования (Bash, PHP) обрамление строковых констант двойными кавычками позволяет вставлять внутрь этих строковых констант динамически вычисляемые переменные: пользуйся этим. 

# 9. Требования к твоим правкам моего программного кода
Если я даю тебе программный код для внесения в него правок, то мой код никогда полностью не переписывай. 
Вместо этого предлагай конкретные точечные пронумерованные по пунктам правки.
Каждая правка должна быть оформлена как заголовок первого уровня (`#`).
