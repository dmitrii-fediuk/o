# 1. ≔
## Syntax
~~~code
A ≔ B
~~~
## Meaning
`B`: некое понятие.
`A`: краткое обозначение для понятия `B`.
Таким образом, `A ≔ B` means that `A` **denotes** `B`.
## Example
`DOR ≔ Florida Department of Revenue` означает, что `DOR` — краткое обозначение для «Florida Department of Revenue».

# 2. ⟨ … ⟩
## Syntax
~~~code
⟨ A ⟩
~~~
## Meaning
It means just `A`.
Таким образом, `⟨ A ⟩ ≡ A`.
## Rationale
Я использую `⟨ A ⟩` вместо `A` в том случае, когда `A` — длинный однострочный текст, и я хочу яснее показать его границы (в частности, когда `A` внутри себя содержит спецсимволы).
## Example
`M ≔ ⟨ Marketplace facilitators (Airbnb/VRBO) ⟩` означает, что `M` — краткое обозначение для «Marketplace facilitators (Airbnb/VRBO)».
###
Этот пример хорошо показывает, почему в данном синтаксисе я использую `⟨ … ⟩` вместо обычных скобок `( … )`: обычные скобки уже являются частью значения, и нестандартными скобками `⟨ … ⟩` я яснее показываю границы значения.

# 3. Multi-line values
## Syntax 1
Стандартный синтаксис Markdown для multi-line values: выделение его границ посредством backticks/tildas. 
### Examples
####
```
Line 1

<…>
```
####
~~~
```
Line 1

~~~
## Syntax 2: ❮ … ❯ 
### Example
❮ Line 1
Line 2
<…>
Last Line ❯
### Rationale 
Я использую `❮ ❯` вместо backticks/tildas для сокращения количества строк в записи: синтаксис `❮ ❯` (в отличие от синтаксиса backticks/tildas) позволяет мне размещать «Line 1» и «Last Line» на тех же строках, где расположены маркеры границ блока: `❮` и `❯`. 

# 4. Metadata: ⟪ … ⟫
## Syntax
~~~code
⟪ M ⟫ A
~~~

## Meaning
`M` — мета-информация об `A`.
## Example
~~~code
⟪ мой ответ клиенту на его письмо `X` ⟫
```
<содержание моего ответа>
```
~~~
В этом примере:
`<содержание моего ответа>` — это текст моего письма клиенту
«мой ответ клиенту на его письмо `X`» — это мета-информация. 
Обрати внимание, что мета-информация не является частью текста моего письма клиенту.
Мета-информация в данном пример указывает назначение теста.

# 6. ⊤
## Syntax
~~~code
⊤ A
~~~
## Meaning
`A` is true (is a fact).
## Rationale
Я обычно использую синтаксис `⊤` на поздних этапах многошагового анализа.
На ранних этапах такого анализа истинность `A` может быть неочевидна.
В процессе анализа может выясниться, что `A` — истинно.
В таком случае я явно пишу `⊤ A`, чтобы ты не тратил своё время на повторный анализ истинности `A` и в дальнейшем анализе считал `A` истинным.

# 9. ⠿{I₁ - Iₙ}  
## Meaning
Такая запись определяет упорядоченное множество посредством точного указания его границ: интервала/диапазона значений.
Это множество включает границы указанного интервала: `I₁` и `Iₙ`. 
## Example
~~~code
⠿{1 - 5} ≡ ⠿{1, 2, 3, 4, 5}
~~~

# 10. ⠿ ⟨ … ⟩

~~~

Такая запись определяет множество / тип / категорию сущностей посредством предиката точного словесного определения `D`.
`D` однозначно определяет полный перечень элементов множества.
## Example
~~~code

~~~ 

# 12. ⠿~
## Syntax
~~~code
⠿~ D
~~~
## Meaning
Такая запись определяет множество посредством неточного, примерного словесного описания `D`.
## Example
~~~code
⠿~ 
```markdown
Social media platforms where users can express their opinions independently of the platform owner
- Reddit
- GitHub
- Сайты сети StackExchange
- Авторитетные сайты отзывов типа Trustpilot, sikayetvar.com и т.п.
```
~~~
Здесь множество «social media platforms» определено неточно, примерно: посредством словесного описания и перечисления некоторых (не всех) элементов в качестве примеров. 


# 8. ᗉ
## Syntax
~~~code
A ᗉ S
~~~
## Meaning
###
`A` — неопределённый элемент `S`.
`S` — некоторое множество/категория сущностей.
###
`S` может быть словесно определено словесно,
###
По смыслу `ᗉ` соответствует неопределённому артиклю «a / an» английского языка.
## Examples
###
`A ᗉ ⟨ balls ⟩` означает, что `A` — это «a ball».
###
`A ᗉ ⟨ ball ⟩` ≡ `A ᗉ ⟨ balls ⟩`

## Следствие
⊤ ⟨ `A` ∈ `S` ⟩

# 14. ᛭Dc
`᛭Dc` — это обозначение `᛭DLᵢ` самого себя.
Другими словами, если текст `᛭DLᵢ` содержит упоминание `᛭Dс` — это значит, что `᛭DLᵢ` упоминает сам себя. 
## Example
Если имя файла `᛭Dᵢ` — `sample.md`, и текст `sample.md` использует обозначение `᛭Dc`, это значит, что `᛭Dc` в данном случае обозначает документ `sample.md`.

# 12. Local Definitions
## 12.1.
~~~code
A[§P] ≔ B
~~~
Означает:
- Для понятия `B` я **временно**, **только в рамках** §`P`, использую обозначение `A`.
- Вне §`P` это правило не применяется: в частности, если до §`P` обозначение `A` имело другой смысл, то после §`P` обозначение `A` снова будет иметь этот смысл.
- По сути, `A[§P] ≔ B` объявляет **локальную переменную** `A` с **областью действия** §`P`.
- В отличие от `A[§P] ≔ B`, `A ≔ B` объявляет **глобальную переменную** `A`.

## 12.2.
~~~code
A[§P₁, §P₂, …, §Pₙ] ≔ B
~~~
Означает, что обозначение `A` имеет значение `B` в контексте ⠿{§`P₁`, §`P₂`, …, §`Pₙ`}.
По сути, это правило аналогично §12.1, но область действия локальной переменной `A` ограничивается не одним пунктом, а множеством пунктов.

## 12.3.
~~~code
A[§P₁-§Pₙ] ≔ B
~~~
Означает, что обозначение `A` имеет значение `B` в контексте ⠿{§P₁-§Pₙ}.
По сути, это правило аналогично §12.1 и §12.2.

# 15. `⧙ ⧘`
## Syntax
```code
A⧙P₁, P₂, …, Pₙ⧘ ≔ ⟨ D1 ⟩ ⸨ 
	D2
⸩
```
## Meaning
###
`A` — сущность, значение которой зависит от параметров `P₁, P₂, …, Pₙ`.
###
`D1` — определение сути `A`.
###
`D2` — определение параметров `A`.
`D2` —  опциональный блок.
Вместо него параметры могут быть определены в `D1`.  

# 16. `⧛ ⧚`
## Syntax
```code
A⧛V₁, V₂, …, Vₙ⧚
```

## Meaning
Такой синтаксис используется в связке с синтаксисом §15.
Он означает сущность `A` при конкретных значениях параметров `P₁, P₂, …, Pₙ`, равных `V₁, V₂, …, Vₙ`

# 20. `ꘖ` (attributes / properties)
## 20.1. Definitions using a global symbol
### 20.1.1. 
#### Syntax
~~~code
ꘖ A ∋ B
~~~
#### Meaning
`B` is an attribute / property of `A`.

### 20.1.2.
#### Syntax
~~~code
ꘖ A ∋ B ≔ ⟪ D ⟫ 
```
V
```
~~~
#### Meaning
~~~code
(ꘖ A ∋ B) AND (B ≔ ⟪ D ⟫ V)
~~~

## 20.2. Definitions using local keys
### 20.2.1. Common rules
####
В §20.1 мы описывали an attribute / property of `A` using глобально доступный затем символ `B`.
####
В §20.2 мы описываем attributes / properties of `A` иначе: using local keys.
####
Эти local keys имеют уникальное значение только в контексте `A`.
####
Вне контекста `A` эти local keys могут иметь другие значения.
Поэтому при сссылке на эти local keys надо обязательно указывать их : `A`.
Конкретный синтаксис для указания контекста описан в §20.2.4. 

### 20.2.2.
#### Syntax
##### Variant 1
~~~code
ꘖ A ∋ ⟨ B ⟩
~~~
##### Variant 2
~~~code
ꘖ A ∋ ⟨ B ⟩ ⟪ D ⟫
~~~

#### Meaning
~~~code
	(`B` is an attribute / property of `A`) 
AND 
	(`B` is a local key, not a standalone entity)
AND 
	(`D` — это комментарий, поясняющий роль `B` в `A`)	
~~~

### 20.2.3.
#### Syntax
~~~code
ꘖ A ∋
```toml
'B1' = 'V1'
'B2' = 'V2'
`B3` = 'V3'
<…>
``` 
~~~
#### Meaning
~~~code
ꘖ A ∋ ⟨ B1 ⟩ ≔ V1
ꘖ A ∋ ⟨ B2 ⟩ ≔ V2
ꘖ A ∋ ⟨ B3 ⟩ ≔ V3
<…>
~~~

### 20.2.4.
#### Syntax
~~~code
A「B」
~~~
#### Meaning
#####
Таким способом мы ссылаемся на local key `B`, определённый ранее в качестве attribute / property of `A` посредством синтаксиса §20.2.2 или §20.2.3.  
#####
Требования к `A「B」` распространяются и все подразделы внутри `A「B」` (например, `A「B」「С」`).
