```
$Article:  Название статьи `A`
$Num_Pages: Количество страниц в статье `A`
```

## 1.
```
Def(A, B)
```
будет обозначать, что для понятия `B` я использую обозначение `A`.

## 2.
```
Def_C(`P`, A, B)
```
будет обозначать `Def(A, B)` в контексте пункта `P`.
Вне пункта `P` это правило не применяется.
Если до пункта `P` обозначение `A` имело другой смысл, то после пункта `P` обозначение `A` снова будет иметь этот смысл.
По сути, `Def_C` объявляет локальную переменную `A` с областью действия `P`.
В отличие от него, `Def` объявляет глобальную переменную `A` (значение которой действует для всех пунктов, кроме `P`).

## 3.
```
Def_Set(S, I)
```
будет обозначать, что `S` — множество всех возможных `I`.

## 4.
```
Def_I(I, S)
```
будет обозначать, что `I` — элемент множества `S`.

## 5.
Def(`A`, «Статья "$Article"»).

## 6.
Def(`Pi`, «страница `A`»).

## 7.
В `A` $Num_Pages страниц.

## 8.
Каждая `Pi` содержится в своём отдельном файле PNG.
Def(`Ii`, «файл PNG для `Pi`»).

## 9.
Def(`Oi`, «порядковый номер `Pi` в `A`»).

## 10.
`Oi` начинаются с `1`.
При этом `Oi` имеют leading zero, если $Num_Pages > 9.

## 11.
Имя файла каждого `Ii` имеет формат: 
```
`Oi`.png
```
### 11.1.
Пример при $Num_Pages ≤ 9:
`1.png`, `2.png`, `3.png`, …
### 11.2.
Пример при $Num_Pages > 9:
`01.png`, `02.png`, `03.png`, …

## 12.
Таким образом, полный текст `A` состоит из $Num_Pages последовательных `Ii`.

## 13.
Я прикрепил к этому запросу следующие `Ii`: `STUB-1.png`, `STUB-2.png`.

## 14.
Def(`F`, «фрагмент `A` раздела "STUB" от пункта STUB и до пункта STUB»).

## 15.
Полностью процитируй `F`.

## 15
Результатом должен стать полный, точный, дословный текст `F`.

## 16.
До и после `F` ничего не пиши.
Например, не пиши фразы типа «The requested fragment…».

## 17.
Тебе запрещено цитировать `F` частично, не полностью.
