# ⦿
## Syntax
<code>⦿ <var>S</var></code>
## Meaning
Конкретный элемент of the <var>`S`</var>.
## Parameters   
- <var>`S`</var>: is a `ꑌ⠿`.
## Rationale
Я использую такой синтаксис, чтобы делать утверждения об одном конкретном (не произвольном) элементе <var>`S`</var>.
## Clarification
Семантически в естественном английском языке это соответствует использованию определённого артикля «the» совместно с термином вроде «element»: «the element».
## Какой элемент я имею в виду?
Сам по себе синтаксис `⦿ S` не содержит конкретики, какой именно элемент я имею в виду.   
Эту конкретику я выражаю отдельно одним из 2 способов:
### Способ 1
В дальнейшей части моего запроса / документа.
#### Example
~~~code
`ꆜ` ≔ ⦿ ⠿ ⟨ потенциальный клиент на Upwork ⟩
Информация о `ꆜ`: <…>
~~~
### Способ 2
Используя мета-данные.
#### Example
~~~code
`I⠿` ≔ ⠿ ⟨ приложенный к этому запросу файл ⟩
`I₁` ≔ ⟪ filename: `1.png` ⟫ ⦿ `I⠿` 
~~~
## `⦿` как бинарный оператор
<code><var>A</var> ⦿ <var>S</var></code> ≡ <code><var>A</var> ≔ (⦿ <var>S</var>)</code>
### Rationale
Используя подобный сокращённый синтаксис, я одним оператором `⦿` делаю сразу 2 вещи: делаю утверждение `⦿ S` и ввожу краткое обозначение <var>`A`</var> для этого утверждения.
### Examples
####
~~~code
`ꆜ` ⦿ ⠿ ⟨ потенциальный клиент на Upwork ⟩
Информация о `ꆜ`: <…>
~~~
####
~~~code
`I⠿` ≔ ⠿ ⟨ приложенный к этому запросу файл ⟩
`I₁` ⟪ filename: `1.png` ⟫ ⦿ `I⠿` 
~~~
### Clarification
По смыслу `⦿` соответствует определённому артиклю «the» английского языка.
