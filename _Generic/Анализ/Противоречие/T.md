# 2. `᛭T`
Действуй пошагово:
##
Определи и систематизируй `C↯⬆⠿`.
##
Определи и систематизируй `C↯⬇⠿`.
##
Проанализируй `C↯⠿`.
В итоге для каждого `C↯ᵢ` выяви:
- Доводы за `Pⰳ(C↯ᵢ)`
- Доводы против `Pⰳ(C↯ᵢ)`
## 
Оцени `Pⰳ(C↯ᵢ)` по шкале от 0 до 100:
- 0 означает, что ты полностью уверен, что `C↯ᵢ` ложна.
- 100 означает, что ты полностью уверен, что `C↯ᵢ` истинна.
##
Предоставь `R`.

# 3. Требования к `R`
##
В `R` должно быть ровно 3 нумерованных раздела (`C↯ᵢ-S⌖1`) 2-го уровня (`##`):
- для `C↯⬆⠿`
- для `C↯⬇⠿`
- для итогового вердикта

##
Каждый `C↯ᵢ` оформляй посредством Markdown как нумерованный подраздел (`C↯ᵢ-S⌖2`) 3-го уровня (`###`) внутри соответствующего `C↯ᵢ-S⌖1`.

## 
Внутри `C↯ᵢ-S⌖2` должны присутствовать следующие нумерованные подразделы, оформленные посредством Markdown как разделы 4-го уровня (`####`):
- Суть
- Оценка
- Доводы за
- Доводы против

## 
### 
Каждый абзац `C↯ᵢ` должен содержать ровно одно предложение.
### 
Между абзацами `C↯ᵢ` не должно оставаться пустых строк.
