## 1. `≔`
### 1.1.
```code
A ≔ B 
```
means that `A` **denotes** `B`.

### 1.2.
~~~code
A ≔
```
B
```
~~~
равнозначно `A ≔ B` и используется, когда `B` — многострочный текст.


## 3. `⊢`
~~~code
A ⊢ B
~~~

## 4. `⊤`
### 4.1.
~~~code
⊤ B
~~~
means that `B` is true (is a fact).

### 4.2.
~~~code

~~~

```
(⊤ `B`) AND (`Rs` are the reasons why `B` is true)
```

### 4.3.
~~~code

```
B
```
~~~
means:
```code
(`A` ≔ `B`) AND (⊤ `B`).
```

### 4.4.
~~~code
A ≔⊤⟦Rs⟧
```
B
```
~~~
means:
```code
(`A` ≔ `B`) AND (⊤⟦Rs⟧ B).
```

## 8. Local Definitions
### 8.1.
~~~code
A[§P] ≔ B
~~~
means:
```markdown
- Для понятия `B` я **временно**, **только в рамках** §`P`, использую обозначение `A`.
- Вне §`P` это правило не применяется: в частности, если до §`P` обозначение `A` имело другой смысл, то после §`P` обозначение `A` снова будет иметь этот смысл.
- По сути, `A[§P] ≔ B` объявляет **локальную переменную** `A` с **областью действия** §`P`.
- В отличие от `A[§P] ≔ B`, `A ≔ B` объявляет **глобальную переменную** `A`.
```

### 8.2.
~~~code
A[§P₁, §P₂, …, §Pₙ] ≔ B
~~~
имеет значение, **аналогичное §8.1** , только в этом случае обозначение `A` имеет значение `B` в контексте **не одного пункта** §`P`, а **всех** пунктов [§`P₁`, §`P₂`, …, §`Pₙ`], **перечисленных** в квадратных скобках.

### 8.3.
~~~code
A[§P₁-§Pₙ] ≔ B
~~~
имеет значение, **аналогичное §8.2**, только в этом случае я в квадратных скобках не перечисляю все пункты §`P₁`, §`P₂`, …, §`Pₙ`, а вместо этого указываю их **диапазон**.
Обозначение `A` имеет значение `B` в контексте **всех** пунктов этого диапазона, **включая** его **границы**: §`P₁` и §`Pₙ`.


## 9. `≔⠿`
### 9.1.
~~~code
S ≔⠿ {I₁, I₂, …, Iₙ}
~~~
будет обозначать, что `S` — множество, заданное точным перечислением всех его элементов: {`I₁`, `I₂`, …, `Iₙ`}.

### 9.2.
~~~code
S ≔⠿ {I₁-Iₙ}
~~~
означает, что `S` — множество, заданное интервалом (диапазоном) его значений.
`S`, в том числе, включает границы указанного интервала: `I₁` и `Iₙ`.

### 9.3.
~~~code
S ≔⠿ (D)
~~~
означает, что `S` — это множество, и `D` — это неформальное (словесное) определение/описание элементов `S`.

### 9.4.
~~~code
S ≔⠿
```
D
```	
~~~
равнозначно `S ≔⠿ (D)` и используется, когда `D` — многострочный текст.

### 9.5.
~~~code
S ≔⠿ (D)
```yaml
- I₁
- I₂
- …
- Iₙ
```	
~~~
означает: (`S ≔⠿ (D)`) AND ({`I₁`, `I₂`, …, `Iₙ`} — примеры некоторых (не всех) элементов `S`) .

## 10.

### 10.2.
~~~code

~~~

```code

```

## 11. `≔†`
~~~code

~~~

```code
(`A` ≔ `B`) AND (`B` is a **problem** to me).
```
